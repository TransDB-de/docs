# Version 3.0.0

**Veröffentlicht:** 08. Juli 2026  
**Zusammenfassung:** Anpassung der Website an das neue C#-Backend, umfangreiche Verbesserungen der Verwaltungsfunktionen und neue Moderationswerkzeuge.


## Das große Update: Neues Backend

Die wohl größte Änderung in dieser Version ist im Hintergrund passiert: Trans\*DB nutzt jetzt ein komplett neues Backend, das in **C#/.NET** geschrieben ist.

Das alte System basierte auf TypeScript und war durch das schnelllebige JavaScript-Ökosystem zunehmend schwer zu warten. Das Framework wurde *deprecated* und ein Rewrite stand ohnehin an. Die neue Basis auf **ASP.NET Core** ist stabiler, zuverlässiger und leichter zu warten.

Die Website wurde angepasst, um mit dem neuen Backend zusammenzuarbeiten. Daneben gab es aber auch eine Reihe von neuen Funktionen und Verbesserungen.


## Neue Funktionen & Verbesserungen

### Einträge zurückziehen

Nutzer\*innen können jetzt einen eingereichten Eintrag selbstständig zurückziehen. Auf der Bestätigungsseite nach dem Einreichen gibt es dafür einen Button. So müssen Einträge nicht erst manuell vom Team gelöscht werden, wenn man seine Meinung ändert.

### Verbesserte Duplikat-Erkennung

Wenn ein Eintrag eingereicht wird, prüft das System automatisch, ob es sich möglicherweise um ein Duplikat handelt. Falls ja, wird dies direkt bei der Einreichung angezeigt. In Kombination mit der neuen Rückziehfunktion reduziert das den Moderationsaufwand erheblich.

### Aktivitätsprotokoll für Einträge

Das Admin-Panel verfügt jetzt über ein vollständiges Aktivitätsprotokoll für jeden Eintrag. Alle Änderungen werden erfasst und können kommentiert werden. Das ist besonders nützlich, um nachzuvollziehen, warum ein Eintrag beispielsweise gesperrt wurde.

### Erweiterter Eintrags-Status

Einträge haben jetzt ein neues Feld `archiviert`. Das ermöglicht eine sanfte Löschung (*Soft-Delete*): Wenn die Inhaber\*in eines Eintrags dessen Entfernung wünscht, wird der Eintrag archiviert statt endgültig gelöscht. So lässt sich das später erkennen, falls derselbe Eintrag erneut eingereicht wird.

### Komplett überarbeitetes Verwaltungs-Panel

Die gesamte Verwaltungsoberfläche wurde überarbeitet, um sie benutzerfreundlicher und vielseitiger zu gestalten.

### Neuer CAPTCHA-Schutz

Die alten CSRF-Tokens wurden durch eine selbstgehostete, unsichtbare CAPTCHA-Lösung ersetzt. Das schützt die API besser vor unbefugtem, automatisiertem Zugriff.

Verwendeter Service: https://trycap.dev/

### Aktualisierte Attribute & Angebote

Die Kategorien, Attribute und Angebote für Einträge wurden überarbeitet und erweitert. Es stehen jetzt noch passendere Auswahlmöglichkeiten für die verschiedenen Eintragsarten zur Verfügung.

### Neuer Geocoding-Service

Das Backend nutzt jetzt den neuen **Trans\*DB Geocoding-Service** mit aktuellen Geodaten und verbesserter Genauigkeit für die Umkreissuche.

Dieser Geocoding-Service wurde übrigens als Open-Source Projekt veröffentlicht:
https://github.com/TransDB-de/geocoding

### Kategorie "Gutachter:innen" entfernt

Die Kategorie *Gutachter\*innen* (Surveyors) wurde entfernt. Diese existierte nur, um Gutachten für die alte Namensänderung nach dem TSG (Transsexuellengesetz) zu verifizieren. Dieses Gesetz wurde inzwischen durch das SBGG (Selbstbestimmungsgesetz) abgelöst, sodass diese Gutachter\*innen nicht mehr benötigt werden.
